# TicTacToe
Tic-Tac-Toe: The Matrix Logic EngineA Python-based Tic-Tac-Toe game where a human player challenges an AI built entirely on NumPy matrices and nested conditional logic. Instead of using a standard minimax recursion, this model utilizes a hard-coded decision tree to evaluate the board state and determine the optimal counter-move.🚀 OverviewThe project demonstrates the power of matrix manipulation and logical branching. By treating the game board as a $3 \times 3$ coordinate system, the AI scans for winning patterns, defensive necessities, and strategic openings.Key FeaturesNumPy Integration: The board is managed as a 2D NumPy array, allowing for efficient slicing and state checking.Rule-Based AI: The model follows a "Tree Search" philosophy implemented via if-else statements to prioritize moves (Win > Block > Center > Corners).Fixed Turn Order: The player always takes the first move, challenging the AI to defend and counter-attack from a disadvantage.🛠️ Tools & TechniquesLanguage: Python 3.xLibraries: NumPyLogic Structures: * Matrix slicing (e.g., board[0, :] for rows, board.diagonal())Conditional branching for state evaluation.Boolean masking to identify empty cells.🎮 How it Works1. The Board RepresentationThe game uses a $3 \times 3$ matrix where:0: Empty cell1: Player (X)2: AI Model (O)2. The Decision HierarchyThe AI evaluates the board in the following order of priority:Win: Can I place a third "2" in any row, column, or diagonal?Block: Is the player one move away from winning? (Block the "1"s).Strategize: Occupy the center or strategic corners to force a draw or setup a future win.
